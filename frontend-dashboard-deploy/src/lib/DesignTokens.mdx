# Design Tokens

Design Tokens are the visual design atoms of the MorningAI design system. They define the fundamental visual elements such as colors, typography, spacing, and more.

## Overview

Our design token system is centralized in `public/tokens.json` and automatically applied to all components through the `.theme-morning-ai` CSS class scope.

## Usage

### In CSS

```css
.my-component {
  color: var(--color-primary);
  font-size: var(--font-size-base);
  padding: var(--spacing-md);
  border-radius: var(--border-radius-md);
  box-shadow: var(--shadow-sm);
}
```

### In Tailwind

Design tokens are automatically integrated with Tailwind CSS through the configuration.

```jsx
<div className="text-primary text-base p-4 rounded-md shadow-sm">
  Content
</div>
```

### In JavaScript

```javascript
import tokens from '@/public/tokens.json';

const primaryColor = tokens.colors.primary;
const baseFontSize = tokens.typography.fontSize.base;
```

## Best Practices

1. **Always use design tokens** instead of hard-coded values
2. **Maintain consistency** across all components and pages
3. **Use semantic colors** (success, warning, error) for meaningful UI states
4. **Follow the spacing scale** for consistent rhythm
5. **Test color contrast** to ensure WCAG AA compliance (4.5:1 for normal text)
